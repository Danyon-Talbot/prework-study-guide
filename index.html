Prework Study Guide

HTML

  • The head element contains information about the webpage
  • The meta element defines metadata such as character set, description, keyowrds, author, and viewport
  • The charset element is used to define what characters and symbols can be used on the page
  • The base element defines the base URL for a webpage
  • The style element defines the CSS styles associated with a website
  • The link element connect and external resource to the HTML document
  • The script element embend client-side scripts in the HTML code
  • The body element represent the visible content shown to the user
  • !DOCTYPE declates what document type the browser should expect when the document is opened. This is placed at the very top of the code
  • Following !DOCTYPE should be the opening tag, this represents the root, or start, of our HTML document

CSS

  • A margin indicates how much space we want around the outside of an element
  • A padding indicates how much space we want around the content inside an element
  • You can combine multiple selectors into a single CSS rule using a comma, allowing the declarations to affect both selectors
  • Do Not Repeat Yourself (DRY) is a coding procedure that encourages developer to reuse or share pieces of code to reduce the number of lines of code that need to be written or maintained
  • "inline CSS" styles apply to a specific HTML tag, using a style attribute with a CSS rule to style a particular element only
  • "internal CSS" style sheets contains CSS rules for the webpage in the head section of the HTML file. The rules only apply to that page
  • "external CSS" style sheets are the preferred style sheet. In most cases they are separate from the main HTML file and can be linked to a desired HTML file

Git

  • git status: Checks what branch we are currently on
  • git checkout -b [branch-name]: Creates a new branch and switches to it
  • git add [file]: Adds a file as it is now to the next commit stage. Use -A to commit all current files
  • git reset: Unstages a file whilst retaining current changes
  • git commit -m [descriptive message]: Commits all currently staged content as a new commit snapshot. -m adds a descriptive message
  • git pull: Fetch and merge any commits from the tracking remote branch
  • git push: Transmit local branch commits to the remote repository branch
